ReeveScript, The All-In-One Compiler, Library, Math Calculator

Reeve or, ReeveScript, Is a Library made to ease the development process by predefining functions that would take hours to make in raw C++.

RS/Reeve Aims to Allow People of all mind spaces to learn to code with no effort at all.

How does it work?

ReeveScript is a Library, For In-Folder Development you'd usually type this.

#include <reeve.h>

 

If you're using the latest edit via git clone then you'd do this.

#include "reeveforvs/api/src/reeve.h"

Reeve.h Compiles all Components of ReeveScript So you could save the trouble of 1000 includes.

Information

  • ReeveScript Uses C++ for its functionality.
  • Reeve Contains a Python API Wrapper for Writing Scripts in python. More information Here
  • PyReeve Gets Updates a couple days after The ReeveScript API Does.

 

Why should you learn Reeve API Script/ReeveScript?

The Reeve Language Compiles code. Except it is not a compiler, By compile it means that it takes C++ Functions and bundles them up to make it easier.

What are the currently available Languages Reeve Supports

  • Python
  • C
  • C++
  • Lua

 

Documentation (You can find the Web Documentation here)

ReeveScript is a high-level Library as it is able to utilize some of the big systems used by languages such as Python, And Ruby.

To program in Reeve You have to understand the basic Rules.

Rulebook Page 1

  • All Functions end with ;
  • You must define a Class Instance before you can use it.
  • You can either have a File in the Same Directory as Reeve.h OR have it inside of the api folder OR Have it as a seperate folder from your project.
  • Every Variable Storage is a Vector meaning you could add Variables through Functions. (INTRODUCED IN 1.12)
  • You can contribute to the github here

Rulebook Page 2

  • You are able to review the source
  • The bin.exe Always Makes the directories in the C Drive.
  • ReeveScript Passes Through Three Languages
  • You can make your own Reeve API Wrapper Just by defining a static C Method function at the beginning!

Scripting 1.2.0

You've Finished reading the rule book, Now for the easy part.

#include "api/src/reeve.h" // Include Reeve Compiler

int EXEC() // Integrated Execute Function.

    Wos32 y; // Define Windows32 Class Instance Variable
    y.writeln("This is a Test.!");
    y.writeln("ReeveScript is a Basic Library Made to ease Development.");
    y.writeln("There's also A BUNCH of modifications to help you in developing games.");
    y.pm("Isn't That pretty Cool? : ");
    y.writeln("There's also Support for languages Like Python And Lua, I Think thats kinda cool!"); 
}

Scripting 2.0

This part goes in order from Finished Implementing to TODOs.

 

Wos32 Lib (IO) (OUTDATED)

  • pm("prompt: ") -- Prompts the user using the user string inside of the function.
  • help() -- Help Function.
  • writeln("Foo") -- Prints Foo.
  • read_file_from_string("File to read") -- Reads the whole file if found
  • intro_sequence("Your Name Here") -- Plays the Intro Sequence With Your Name in it.
  • storecache() -- Caches the data of the Application Current State.

Wofilestream (FS) (NOT UPDATED)

  • MakeFile("file.ext") -- Creates a file. Classic!
  • addif() -- Constant Checker for files.
  • readPrintIf() -- Reads a file, Prints if what is inside a file is true.
